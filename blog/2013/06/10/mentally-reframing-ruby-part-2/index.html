Mentally Reframing Ruby – Part 2
Jun 10th, 2013

What the What with the Gmail

After my first experiment with Ruby and Arduino, I wanted to try writing a sketch that would build off of the code I had written from the blinking LED test. Using the blinking LED as an indicator, I thought it would be interesting to see if I could get my Ardunio talking with Gmail to alert me if I had any unread inbox messages. After doing some research, I found a gem called Gmail that allows me, through Ruby, to do just that.

Requiring the Right Gems

Here we’re going to set up the sketch with the gems we’ll be running.

Board in the USA

Dino requires us to define our board, tell it that we’ll be inputing and outputing data, and tell it that we’re using a serial connection (since my Ardunio is hooked up via USB). Next, we’ll be defining our LED variable, which should be easy since it was in the previous sketch. Note: Dino also offers a good directory full of example code for setting up any given component.

Setting up dino

You’ve Got Mail

In the next few lines of code we: call on Gmail (through the gmail gem) to let us access our inbox, check if our inbox is empty, set the led variable to a resting state, and define a variable equal to the number of unread emails.

Ruby Slippers

Now that we have access to our inbox, we can finish up this sketch by writing an if statement to blink our led when our inbox is not at zero.

Next Steps

This is a poorly written sketch, but for the purposes of a dry run it did its job. I’m hoping to take this further by writting in a delay and loop so that the sketch runs every 5-10 mins. Another feature I’d like to add is a rgbLED so that I can light up a red LED when the inbox is full and a green LED when the inbox is empty.
